# **🛒 Java Commerce Project**

객체 지향 설계를 기반으로 구현한 콘솔 기반 커머스 플랫폼입니다.
상품 관리, 카테고리, 장바구니, 주문, 관리자 모드를 단계적으로 구현하며
Java OOP 개념과 예외 처리, 캡슐화를 학습하는 것을 목표로 합니다.

## **📌 프로젝트 개요**

이 프로젝트는 Java의 객체 지향 설계 원칙을 적용하여
실제 커머스 플랫폼의 핵심 기능을 콘솔 환경에서 구현한 학습용 프로젝트입니다.

메인시스템(CommerceSystem) 출력담당

상품(Product) 관리

카테고리(Category) 구조

장바구니(Cart) & 주문(Order)

관리자 모드 (MasterService) 

고객(Customer)

고객등급(CustomerGrade)

예외 처리 및 입력 검증

## **💡 핵심 학습 포인트**

객체 지향 설계 (SRP, 책임 분리)

클래스 간 협력 구조 설계

예외 처리 (try-catch, 커스텀 예외)

컬렉션 활용 (List, ArrayList)

입력 검증 및 사용자 인터랙션 설계

캡슐화 & 정보 은닉

# 필수기능

## **STEP 1. 상품 목록 출력**
목표

Product 클래스를 생성하여 여러 상품을 관리

List<Product>를 활용한 상품 목록 출력

**주요 구현**

Product 클래스: 상품명, 가격, 설명, 재고수량 필드

ArrayList<Product>로 여러 상품 관리

반복문을 통해 상품 목록 출력

## **STEP 2. CommerceSystem으로 흐름 제어**
**목표**

모든 입력 및 로직을 CommerceSystem 클래스에서 관리

start() 메서드로 프로그램 진입점 분리

**주요 구현**

main 함수에서는 CommerceSystem 객체만 생성

실제 비즈니스 로직은 start() 내부에서 처리

## STEP 3. Category & Customer 도입
**목표**

상품을 카테고리 단위로 관리

고객 정보를 객체로 분리

**주요 구현**
Category

카테고리 이름 + List<Product> 관리

전자제품 / 의류 / 식품 구조

Customer

고객명, 이메일, 등급, 누적 주문 금액 필드

고객 등급은 누적 주문 금액 기준으로 결정

## STEP 4. 캡슐화 적용
**목적**

모든 클래스의 필드를 private으로 보호

Getter / Setter를 통한 접근

**효과**

객체 내부 상태 보호

유지보수 및 확장성 향상

# 도전기능

## 🚀 LV 1. 장바구니 & 주문 기능
**주요 기능**

상품 장바구니 추가

수량 입력 및 재고 검증

장바구니 출력

총 금액 계산

주문 완료 시 재고 차감

## 🛠 LV 2. 관리자 모드

관리자 전용 기능을 MasterService 클래스로 분리하여 관리

관리자 인증

비밀번호 입력 (3회 실패 시 메인 복귀)

상품 관리 기능

상품 추가

상품 수정

상품 삭제

같은 카테고리 내 중복 상품명 검증

삭제된 상품은 장바구니에서도 제거
